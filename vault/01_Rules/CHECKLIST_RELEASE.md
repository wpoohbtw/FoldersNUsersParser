# Release Checklist

Этот чеклист используется только для RELEASE TASK:
- финальная проверка;
- подготовка к деплою;
- полный аудит;
- проверка перед релизом.

Для QUICK FIX и обычных локальных задач этот чеклист полностью не выполнять.

## 1) Перед началом задачи
1. Зафиксировать скоуп: что меняем и что точно не меняем.
2. Свериться с `vault/02_Patterns/*` для похожего кейса.
3. Проверить, затрагивает ли задача UI, backend, API, БД, кодировку.

## 2) Перед сдачей
1. Scope-check:
   - `git diff --name-only` содержит только ожидаемые файлы.
   - Нет побочных правок в соседних модулях.
2. Кодировка и моджибейка:
   - `rg -n "РџС|РќР|РЎР|СЃ|С‚|�" src backend`
   - визуально проверить измененные тексты.
3. Базовая функциональная проверка:
   - если трогали API: быстрый smoke проблемных эндпоинтов;
   - если трогали UI: проверить критичный пользовательский сценарий.
4. Паттерны:
   - сравнить с эталоном в `vault/02_Patterns/PAGE_STANDARD_ACCOUNTS_MANAGER.md` (для похожих экранов).

## 3) После сдачи
1. Обновить `vault/03_Projects/CHANGELOG_AI.md`.
2. Если появился новый устойчивый подход - добавить в `vault/02_Patterns/COMPONENT_PATTERNS.md`.
3. Если поменялись контракты API - обновить `vault/03_Projects/API_MAP.md`.
4. Если поменялась терминология - обновить `vault/03_Projects/GLOSSARY.md`.

## 4) Красные флаги (стоп-проверка)
1. В diff есть файлы вне скоупа.
2. В UI или API-ошибках появились строки с артефактами кодировки.
3. Изменен эталонный паттерн без отражения в vault.
4. Изменены API-контракты, но карта API не обновлена.
